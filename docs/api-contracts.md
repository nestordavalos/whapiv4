## Authentication

### JWT Session Auth (`isAuth`)

Used by internal UI endpoints. Reads `Authorization: Bearer <token>` header.

- Validates JWT against `JWT_SECRET`
- Checks session expiry against `Setting.userSessionTimeout`
- Sets `req.user = { id, profile }` on success
- Emits `session:expired` WebSocket event on timeout

### API Token Auth (`isAuthApi`)

Used by external API endpoints (`/api/*`, `/api/v1/*`). Reads `Authorization: Bearer <apiToken>` header.

- Validates token against `userApiToken` setting stored in DB
- Lighter-weight auth for programmatic access
