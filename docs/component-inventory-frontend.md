### Route Configuration

Routes are defined in `src/routes/index.jsx` with two categories:

- **Public routes** (`/login`, `/signup`) — accessible without auth
- **Protected routes** — wrapped in auth check, redirect to `/login` if not authenticated
- **Admin-only routes** (`/connections`, `/users`, `/quickAnswers`, `/Settings`, `/Queues`, `/Tags`, `/api`, `/apidocs`, `/apikey`, `/queue-integrations`) — require `user.profile === "admin"`

## Custom Hooks

### useAuth

```
File: hooks/useAuth/index.jsx
Returns: { isAuth, user, loading, handleLogin, handleLogout }
```

Manages authentication lifecycle:
- Auto-refreshes token on mount via `POST /auth/refresh_token`
- Sets up axios interceptors for Bearer token injection
- Auto-refreshes on 403, logs out on 401
- Listens to `user` socket events for profile updates

### useTickets

```
File: hooks/useTickets/index.jsx
Params: { searchParam, pageNumber, status, date, showAll, queueIds, tags, whatsappIds, userIds, userId, withUnreadMessages }
Returns: { tickets, loading, hasMore, count }
```

Paginated ticket fetching with debounce (200ms for closed, 500ms for others). Appends on pagination, replaces on page 1.

### useWhatsApps

```
File: hooks/useWhatsApps/index.jsx
Returns: { whatsApps, loading }
```

Fetches all WhatsApp connections and maintains real-time state via `useReducer` + socket events (`whatsapp`, `whatsappSession`). Actions: LOAD, UPDATE, DELETE, UPDATE_SESSION.

### useDashboard

```
File: hooks/useDashboard/index.jsx
Returns: { find } → find(params) returns dashboard data
```

Calls `GET /dashboard` with date range and filter parameters.

### useQueues

```
File: hooks/useQueues/index.jsx
Returns: { findAll } → findAll() returns all queues
```

Calls `GET /queue`.

### useLocalStorage

```
File: hooks/useLocalStorage/index.jsx
Params: (key, initialValue)
Returns: [storedValue, setValue]
```

React state synced with `localStorage`. Used for persistent UI preferences (audio speed, sign toggle, sidebar state).

## API Services

### Axios Instance — `services/api.jsx`

Base Axios instance with:
- `baseURL` from `getBackendUrl()` (reads `config.json`)
- `withCredentials: true` (for cookie-based refresh tokens)
- All components import this instance for API calls

### Socket.IO — `services/socket-io.jsx`

| Function | Description |
|----------|-------------|
| `connectToSocket()` | Singleton Socket.IO connection with JWT auth. Validates token expiry via `jwt-decode`. Handles `connect_error` (expired → logout) and `session:expired` → redirect. Uses websocket transport with infinite reconnection. |
