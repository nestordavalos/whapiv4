## Quick Start (Docker)

```bash
# 1. Clone the repository
git clone <repo-url> && cd whapiv4

# 2. Copy environment files
cp .env.example .env
cp backend/.env.example backend/.env
cp frontend/src/config.json.example frontend/src/config.json

# 3. Edit .env with your settings (DB passwords, JWT secrets, URLs)
# Edit backend/.env with your backend config
# Edit frontend/src/config.json for branding/theme

# 4. Build and start all services
docker-compose up -d --build

# 5. (Optional) Add Browserless Chrome for headless WhatsApp
docker-compose -f docker-compose.yaml -f docker-compose.browserless.yaml up -d --build

# 6. (Optional) Add phpMyAdmin for DB management
docker-compose -f docker-compose.yaml -f docker-compose.phpmyadmin.yaml up -d
```

Services will be available at:
- **Backend**: `http://localhost:${BACKEND_PORT}` (default 8080)
- **Frontend**: `http://localhost:${FRONTEND_PORT}` (default 3333)
- **phpMyAdmin**: `http://localhost:${PMA_PORT}` (default 9000)

> Migrations run automatically on backend container startup.

## Deployment

### Docker (Recommended)

1. Configure `.env` files (root, backend, frontend)
2. `docker-compose up -d --build`
3. Migrations run automatically
4. SSL: Place certs in `ssl/certs/` for HTTPS

### VPS Manual (PM2 + Nginx)

1. Install Node 16, Chrome, MySQL/MariaDB
2. Build backend: `npm install && npm run build`
3. Run migrations: `npx sequelize db:migrate && npx sequelize db:seed:all`
4. Start with PM2: `pm2 start dist/server.js --name backend`
5. Build frontend: `npm install && npm run build`
6. Serve with PM2: `pm2 start server.js --name frontend` (uses Express static server)
7. Configure Nginx reverse proxy for both services
8. SSL with Certbot/Let's Encrypt

## Troubleshooting

### Common Issues

| Issue | Solution |
|-------|---------|
| Chrome not found | Set `CHROME_BIN` env var or use Browserless Docker compose |
| WhatsApp QR not showing | Check Chrome binary, verify `CHROME_ARGS` includes `--no-sandbox` |
| CORS errors | Ensure `FRONTEND_URL` in backend `.env` matches actual frontend URL |
| DB connection refused | Check `DB_HOST`, `DB_PORT`, `DB_USER`, `DB_PASS` in backend `.env` |
| S3 upload fails | Verify S3 credentials, bucket permissions, `STORAGE_S3_FORCE_PATH_STYLE` for MinIO |
| Socket.IO disconnects | Check proxy configuration, ensure WebSocket upgrade is allowed in Nginx |
| Frontend blank page | Verify `VITE_BACKEND_URL` points to accessible backend URL |
| Migration errors | Run `npx sequelize db:migrate:undo` then `npx sequelize db:migrate` again |
| PM2 restart fails | Verify `PM2_FRONTEND` and `PM2_BACKEND` env vars match PM2 process names |

### Logs

- **Backend**: stdout/stderr or Pino JSON logs
- **Sentry**: Errors reported if `SENTRY_DSN` configured
- **Docker**: `docker-compose logs -f backend`
- **PM2**: `pm2 logs backend`
